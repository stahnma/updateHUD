API Documentation

MQTTFun Server API Reference

GET /api/systems
Returns a JSON array of all registered systems with their update status and pending updates.
Response
200 OK Returns an array of system summaries
[ { "hostname": "server1.example.com", "architecture": "amd64", "ip": "192.168.1.100", "os": "debian", "os_version": "12", "updates_available": true, "update_status_unknown": false, "last_seen": "2024-01-15T10:30:00Z", "pending_updates": [ { "name": "nginx", "version": "1.25.0", "source": "apt" } ] } ]
GET /api/systems/{hostname}
Returns detailed information for a specific system identified by hostname.
Path Parameters
hostname
The hostname of the system to retrieve (required)
Response
200 OK Returns the system details
{ "hostname": "server1.example.com", "architecture": "amd64", "ip": "192.168.1.100", "os": "debian", "os_version": "12", "updates_available": true, "update_status_unknown": false, "last_seen": "2024-01-15T10:30:00Z", "pending_updates": [ { "name": "nginx", "version": "1.25.0", "source": "apt" }, { "name": "openssl", "version": "3.0.11", "source": "apt" } ] }
404 Not Found System not found
DELETE /api/systems/{hostname}
Deletes a system from the database by hostname.
Path Parameters
hostname
The hostname of the system to delete (required)
Response
200 OK System deleted successfully
{ "status": "success", "message": "System deleted successfully", "hostname": "server1.example.com" }
404 Not Found System not found
400 Bad Request Hostname is required
GET /ws
WebSocket endpoint for real-time system updates. Clients can connect to receive live updates when systems report their status.
WebSocket Protocol
Connection
Upgrade HTTP connection to WebSocket using the standard WebSocket handshake
Messages
Server sends JSON-encoded system update objects when systems report their status
Keepalive
Server sends ping frames every 30 seconds. Client can send "ping" text messages for keepalive
GET /metrics
Prometheus metrics endpoint. Returns metrics in Prometheus exposition format for monitoring and observability.
Response
200 OK Returns metrics in Prometheus text format
Content-Type
text/plain; version=0.0.4; charset=utf-8
Available Metrics
HTTP Metrics
http_requests_total: Total HTTP requests by method, route, and status code
http_request_duration_seconds: HTTP request duration histogram
NATS Metrics
nats_messages_received_total: Total NATS messages received by subject
nats_message_processing_duration_seconds: NATS message processing duration
nats_connection_connected: NATS connection status (1 = connected, 0 = disconnected)
nats_reconnects_total: Total NATS reconnection attempts
Storage Metrics
storage_operation_duration_seconds: Storage operation duration by operation type
storage_operation_errors_total: Storage operation errors by operation type
WebSocket Metrics
websocket_connections_active: Number of active WebSocket connections
websocket_messages_broadcast_total: Total WebSocket messages broadcast
websocket_broadcast_duration_seconds: WebSocket broadcast duration
Business Metrics
systems_monitored_total: Total number of systems being monitored
systems_with_updates_total: Number of systems with pending updates
pending_updates_total: Total number of pending updates across all systems
Example Usage
# Scrape metrics with Prometheus # Add to prometheus.yml: scrape_configs: - job_name: 'mqttfun-server' static_configs: - targets: ['localhost:8080'] # Or view directly in browser curl http://localhost:8080/metrics
Data Models
System
hostname (string): System hostname
architecture (string): System architecture (e.g., "amd64", "arm64")
ip (string): System IP address
os (string): Operating system name (e.g., "debian", "fedora", "macos")
os_version (string): Operating system version
updates_available (boolean): Whether updates are available
update_status_unknown (boolean): Whether update status could not be determined
last_seen (string): ISO 8601 timestamp of last system check-in
pending_updates (array): Array of Update objects
Update
name (string): Package name
version (string): Available version
source (string): Package manager source (e.g., "apt", "dnf", "brew")
